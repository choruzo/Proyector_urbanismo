## 7. Auditoría de datos hardcoded — correcciones P1 ✅

> [!success] Completado — 2026-03-19
> Auditoría post-v0.2. Tres issues P1 detectados y resueltos arquitecturalmente. Fallbacks activos en dev (esperado). Infraestructura lista para producción.

> [!note] Contexto
> Tras cerrar las 6 tareas principales de v0.2, se realizó una auditoría para identificar datos que se presentan como "reales" pero no se obtienen de ninguna fuente externa. Se detectaron 3 issues P1 (críticos) y 5 issues P2 (menores). Los P1 fueron resueltos en esta misma sesión.

### P1-A — Precio base €/m² dinámico (MIVAU SEDAL)

**Problema**: `PRECIO_BASE_2026_EUR_M2 = 1_800.0` era una constante hardcoded de módulo en `initial_load.py`.

**Solución implementada**:
- Nuevo método `ViviendaScraper.get_precio_m2_vivienda_madrid()` en `backend/app/scrapers/vivienda.py`
- Nueva URL en `URLS_ESTADISTICAS["precios_madrid_vivienda_libre"]` → `09110010.XLS` (patrón SEDAL Madrid provincia, serie precios tasación)
- Parser por escaneo de rango `800 < val < 8_000 €/m²` (discrimina automáticamente de recuentos de viviendas)
- `_cargar_valores_suelo()` llama al scraper; si responde → `fuente="mivau_sedal_ipv"`; si falla → `fuente="ine_ipv_estimado"` con fallback `_PRECIO_BASE_FALLBACK = 1_800.0`
- Constante `PRECIO_BASE_2026_EUR_M2` eliminada del scope de módulo

**Estado en dev**: URL `09110010.XLS` devuelve HTTP 404 (fichero no publicado en SEDAL en 2024). El fallback 1.800 €/m² se activa limpiamente. Verificado en logs:
```
WARNING | PRECIO_BASE: MIVAU no disponible; usando fallback 1800.0 €/m²
```
**Pendiente para producción**: identificar la URL correcta de descarga del Excel de precios MIVAU. Candidatas exploradas sin éxito: 15+ URLs `apps.fomento.gob.es/Boletinonline/sedal/09XXXXXX.XLS`. La URL correcta requiere navegación manual por el Portal Estadístico MIVAU.

### P1-B — Coeficientes de precio por barrio desde JSON externo

**Problema**: `COEF_BARRIO = {"GF01": 1.05, ...}` era un dict hardcoded de módulo. Cambiar un coeficiente requería editar código Python y redesplegar.

**Solución implementada**:
- Nuevo fichero `backend/app/data/coef_barrio.json` (cubierto por `COPY . .` del Dockerfile)
- Nueva función `_cargar_coef_barrio() -> dict[str, float]` con `try/except FileNotFoundError`
- Fallback `_COEF_BARRIO_FALLBACK` (mismos valores) si el JSON no existe o es inválido
- Path dinámico: `Path(__file__).parent.parent / "data" / "coef_barrio.json"` → `/app/app/data/coef_barrio.json` en contenedor

**Estado verificado**:
```
INFO | Coef barrio: 12 valores cargados desde coef_barrio.json
```
**Ventaja**: los coeficientes son ahora configurables sin cambios de código. En v0.3 se puede habilitar un endpoint admin para actualizarlos desde la UI o reemplazarlos por valores reales del Registro de la Propiedad.

### P1-C — Geometría real de barrios desde Catastro WFS

**Problema**: `_cargar_barrios()` hacía una llamada WFS de "solo test de conectividad" pero nunca almacenaba geometría en `Barrio.geom`. Siempre `NULL`.

**Solución implementada**:
- Nuevo método `CatastroScraper.get_zonas_catastrales_wfs()` → `CP:CadastralZoning` WFS INSPIRE
- Nueva función libre `_extraer_geometrias_zonas(geojson, barrios_getafe)` con geopandas/shapely:
  - Fuzzy match por primera palabra del nombre de barrio contra propiedades WFS (`label`, `name`, `nombre`, `zone`)
  - Union de polígonos solapados con `shapely.ops.unary_union`
  - Conversión a `SRID=4326;MULTIPOLYGON(...)` para GeoAlchemy2
  - Aproximación de superficie en m² (grados → metros a lat ~40.3°N)
- `_cargar_barrios()` ahora llama a `get_zonas_catastrales_wfs()` y si hay features → `_extraer_geometrias_zonas()` → inserta `WKTElement(wkt, srid=4326)` en `Barrio.geom`
- Con `--force`: actualiza `geom` y `superficie_m2` de barrios existentes si se obtienen datos

**Estado en dev** (Docker sin DNS externo — esperado):
```
WARNING | WFS CadastralZoning no disponible ([Errno -2] Name or service not known)
WARNING | CadastralZoning sin datos; barrios sin geometría (dev esperado)
```
**En producción**: `get_zonas_catastrales_wfs()` alcanzará `www.catastro.meh.es` → hasta 12/12 barrios con geometría MULTIPOLYGON real → habilita el mapa de calor por barrio en v0.3.
